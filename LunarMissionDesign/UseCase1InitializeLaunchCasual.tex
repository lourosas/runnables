\textbf{Initialize Launch--Casual Format}\\
\textit{Main Success Scenario}
\begin{enumerate}
   \item With the System running, the Flight Controller enters the
      Mission Parameters File
   \item System opens the Parameters file
   \item System reads the Parameters file data
   \item System initializes each of its Components by sending the
      Parameters File to each Component
   \item System alerts the Flight Director the Initialization Status
   \item System periodically queries the Components' Status 
      \textbf{or} each Component periodically feeds the System its
      current Status for the Launch duration
   \item System alerts the Stake Holders of its current Status
\end{enumerate}
\textit{Alternative Scenarios}
\begin{enumerate}
   \item[2a] If there is an error in opening the Parameters file,
      then the System alerts the Flight Director of the Error:
      indicating unable to open the Parameters file.
   \item[3a] If System cannot read the Parameters file, then the
      System alerts the Flight Director of the Error:  indicating
      unable to read the Parameters file.
   \item[4a] If the System Components are unable to open the Parameter
      file then the Components alert the System and the System
      alerts the Flight Director
   \item[4b] If the System Components are unable to read the
      Parameter file, then the Components alert the System and
      the System alerts the Flight Director
   \item[6a,7a] The System alerts the Stake Holders of any errors
      discovered.
\end{enumerate}
\textit{Notes/Constraints}
\begin{enumerate}
   \item[*a] The System is made up of components.
   \item[*b] The Components will either alert the System autonomously,
      or the System as a whole will query its components.
\end{enumerate}
